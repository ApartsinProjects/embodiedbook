Worked Example

The following compact probe makes Sample-efficiency advantages and failure modes concrete. It keeps the environment small enough to inspect, but it still records the quantities a real embodied system would need before trusting the method.

# Score short action sequences with a tiny learned-model stand-in.
# Cross-entropy method style search keeps the MPC loop visible.
import numpy as np

rng = np.random.default_rng(7)
start = 0.0
goal = 1.0
horizon = 5
candidates = rng.normal(loc=0.15, scale=0.08, size=(64, horizon))
rollout_end = start + candidates.sum(axis=1)
cost = (rollout_end - goal) ** 2 + 0.02 * np.square(candidates).sum(axis=1)
best = candidates[np.argmin(cost)]
print({"first_action": round(float(best[0]), 3), "best_cost": round(float(cost.min()), 4)})

The output reports the first MPC action and the sequence cost, the two quantities a receding-horizon controller needs before it replans.

Common Failure Mode

The common mistake is to evaluate a component in isolation and assume the closed loop will inherit that score. Embodied systems often fail at the interfaces between perception, representation, planning, control, and evaluation.
