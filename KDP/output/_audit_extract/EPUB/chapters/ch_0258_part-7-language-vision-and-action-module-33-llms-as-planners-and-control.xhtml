"The model did not lie about the object's location. It simply never noticed that the object had moved. Hallucination in the physical world is a synchronization failure, not a language failure."

Big Picture

A robot arm was told to fetch the red cup. Thirty seconds earlier, a person moved it. The arm reached confidently for empty space. The LLM planner had not lied: it had simply never been told the world changed. That gap between a model's remembered state and the actual scene is where most physical task failures live today. As large language models (LLMs) move from writing assistants to robot controllers, their tendency to treat stale context as ground truth becomes a safety issue, not just an accuracy issue. This section shows you how to audit what an embodied agent "remembers," when those memories expire, and how to catch hallucinated state before a gripper acts on it.

LLM memory must be paired with explicit state tracking so that past context helps planning without overriding new sensor evidence.

Once memory and state tracking are treated as one discipline, the design problem becomes a sorting problem.

The practical question is which memories should live as symbolic facts, which should live as scene state, and how hallucinated memories should be caught before action.

Action Is The Test

Embodied memory is only useful if it carries provenance and freshness. A remembered object location with no timestamp is not memory; it is a latent bug.

Theory

Let memory items be facts $m_i = (f_i, c_i, t_i)$ with content, confidence, and timestamp. A planner should reason over a belief state not over free-floating text summaries alone. New observations should update or erase memory items whose confidence is no longer justified.

When the Belief-State Formula Matters

This formalism becomes practically important in three situations: (1) the robot executes a multi-step task spanning minutes, so the scene can change between steps; (2) multiple agents share the same environment, meaning one agent's action silently invalidates another's memory; (3) the LLM plans at a high abstraction level while low-level controllers run at millisecond rates, creating a temporal gap between language-level "knowledge" and physical reality. In simpler single-step fetch tasks the gap rarely matters. In household or warehouse deployments with moving objects and parallel actors, ignoring the belief-state update rule is the primary source of execution failures that look like hallucination but are actually synchronization failures.

Hallucination in embodied tasks often means one of three things: inventing an object or tool, asserting a stale state as current, or carrying a wrong relational fact across scene changes. The fix is rarely 'better prompting' alone. It is usually a better contract between memory, observation, and plan verification.

Before looking at where this goes wrong at scale, it helps to fix the vocabulary: a memory item's confidence is the planner's belief that a fact is still true, and its freshness is how recently that fact was confirmed by a sensor. The worked examples below use both terms, and each empirical result that follows is best read as a report of what happens when freshness is not checked before confidence is trusted.

Two empirical results demonstrate this distinction. The SayCan system (Ahn et al., 2022) grounded language plans in a value function over robot affordances, where the value function is a learned score that estimates how likely a candidate action is to succeed given the robot's current physical situation, which reduced infeasible action selection. Its remaining failure mode was often stale object state. The model would plan to pick up an item whose location had changed since the last observation. EmbodiedBench (Wang et al., 2025) confirms the pattern at scale. On tasks with hidden state changes between replanning steps, even frontier models drop success rates by 30 to 50 percentage points relative to static scenes. A model that solves 80 of 100 static-scene tasks solves only 30 to 56 of the equivalent dynamic-scene tasks, and the sole difference is that an object moved while the planner was not watching. Both results expose the same architectural gap: an LLM that rewrites its plan from scratch each step treats its context window as ground truth, not as a belief that needs freshness gating.

Mechanism

A good memory system separates semantic memory, such as user preference, from dynamic world state, such as object location. The first may persist across episodes; the second should expire quickly or be refreshed from sensors before use.

Worked Example

Code Fragment 1 stores two memories with different freshness and shows how the planner should gate them before use. The example demonstrates why timestamps belong in the memory schema.

# Reject stale world-state memory while keeping durable preference memory.
# Embodied memory should store freshness and source, not just text.
# This keeps old observations from masquerading as current state.
memory = [
{"fact": "user_prefers_blue_mug", "age_s": 600, "durable": True},
{"fact": "red_mug_is_on_counter", "age_s": 45, "durable": False},
]
usable = [m["fact"] for m in memory if m["durable"] or m["age_s"] < 10]
print(usable)
['user_prefers_blue_mug']

The expected output is a memory subset where durable user preferences survive but stale scene claims do not. The point is that embodied memory should grant planning authority only to facts whose lifespan matches the kind of fact they are, not to every retrieved sentence equally.

Code Fragment 1: This gating rule preserves long-lived preference memory while rejecting stale world-state memory. The key lesson is that not all remembered text should have equal planning authority once the physical scene may have changed.

Set the freshness threshold for dynamic world-state facts to no more than one replanning cycle. If your LLM planner runs every 5 seconds, set max_age_s to 5 in a Pydantic MemoryItem model (Pydantic is a Python library that validates data against a declared schema at object-creation time, rejecting malformed records before they reach application logic) and validate it at retrieval time rather than in ad-hoc if checks. Using Pydantic means a missing timestamp raises a ValidationError before the planner ever sees the record, catching the schema violation at write time instead of silently letting an undated fact pass as fresh. For semantic memory such as user preferences, omit the expiry field entirely and use a separate model class so the type system itself encodes the distinction.

Library Shortcut

State stores, graph memories, and vector memories can all hold the facts, but they are only safe in robotics when coupled to freshness metadata and sensor-side verification hooks. The library can manage retrieval; it cannot decide which physical facts are still true.

Practical Recipe

  1. Store memory items with source, timestamp, confidence, and type.
  2. Separate durable preferences from dynamic world-state facts.
  3. Refresh or invalidate dynamic facts before high-consequence actions.
  4. Never let retrieved text bypass a verifier when the action depends on current geometry.
  5. Log contradictions between memory and observation as first-class events.

Algorithm: Freshness-Gated Belief Update for Embodied Memory

Input: Memory store $\mathcal{M} = \{m_i = (f_i, c_i, t_i)\}$ (facts, confidences, timestamps); new observation $o_t$ at time $t$; freshness threshold $\tau$; belief state $b_{t-1}$; planned action $a$

Output: Updated belief state $b_t$; validated memory subset $\mathcal{M}_{\text{valid}}$; contradiction log $\mathcal{C}$

  1. Partition $\mathcal{M}$ into durable facts $\mathcal{M}_D$ (user preferences, invariants) and dynamic facts $\mathcal{M}_W$ (object locations, scene state).
  2. For each dynamic fact $m_i \in \mathcal{M}_W$, compute age $\delta_i = t - t_i$. Retain $m_i$ only if $\delta_i \le \tau$; mark expired facts as stale and exclude them from $\mathcal{M}_{\text{valid}}$.
  3. Apply the Bayesian belief update (revising the probability assigned to each possible world state by combining what was believed before with the new evidence just observed): $b_t = p(s_t \mid o_{1:t},\, a_{1:t-1},\, \mathcal{M}_{\text{valid}})$, giving higher weight to sensor evidence $o_t$ than to any single memory item $m_i$ when they conflict.

Checkpoint

So far: memory is split into durable and dynamic facts (step 1), stale dynamic facts are dropped by an age check (step 2), and the belief state is rebuilt from what survives plus the newest sensor reading (step 3). The remaining steps decide, given that belief state, whether a specific action is allowed to proceed.

  1. For each retained fact $m_i \in \mathcal{M}_{\text{valid}}$, check consistency with $o_t$. If $m_i$ contradicts $o_t$ beyond tolerance $\epsilon$, append $(m_i, o_t, t)$ to contradiction log $\mathcal{C}$ and reduce $c_i \leftarrow \alpha \cdot c_i$ with decay $\alpha \in (0,1)$.
  2. For action $a$ that depends on a geometric or relational world-state fact, require $c_i \ge c_{\min}$ and $\delta_i \le \tau$ before granting planning authority. Reject $a$ and request sensor refresh otherwise.
  3. If a sensor refresh is triggered, issue a perception query $\pi_{\text{verify}}$ targeting the specific object or relation that $a$ depends on, then return to step 2 with the updated observation.
  4. Log the selected action $a$, the supporting facts $\mathcal{M}_{\text{valid}}$, and any contradiction events in $\mathcal{C}$ as a single provenance record before execution.
  5. After execution, record the observed outcome $o_{t+1}$ and update $c_i$ for all facts whose truth value can now be confirmed or refuted by $\nabla$-graded sensor feedback.

Step-Through: Freshness-Gated Belief Update

Trace the algorithm with a tiny example. Set the freshness threshold $\tau = 5$ s, the current time $t = 50$ s, and a confidence floor $c_{\min} = 0.6$. The memory store holds three facts: $m_1 = (\text{user\_prefers\_blue\_mug},\ c=0.9,\ t_1=\text{durable})$, $m_2 = (\text{red\_mug\_on\_counter},\ c=0.8,\ t_2=48\text{ s})$, and $m_3 = (\text{plate\_in\_sink},\ c=0.7,\ t_3=6\text{ s})$. Step 1 partitions: $\mathcal{M}_D = \{m_1\}$, $\mathcal{M}_W = \{m_2, m_3\}$. Step 2 computes ages: $\delta_2 = 50 - 48 = 2$ s, so $m_2$ is retained ($2 \le 5$); the plate fact was observed at $t_3 = 6$ s, so $\delta_3 = 50 - 6 = 44$ s, which exceeds $\tau$, so $m_3$ is marked stale and dropped. Now $\mathcal{M}_{\text{valid}} = \{m_1, m_2\}$. Step 3 builds the belief state from these two facts plus the live observation $o_t$. Suppose the action $a = \text{grasp the red mug}$. Step 5 checks $m_2$: confidence $0.8 \ge 0.6$ and age $2 \le 5$, so authority is granted and the grasp proceeds. Contrast with $a' = \text{grasp the plate}$: $m_3$ is already excluded, so step 5 rejects $a'$ and step 6 issues a verification query $\pi_{\text{verify}}$ at the sink. The arm never reaches for a plate it last saw 44 seconds ago.

Common Failure Mode

The easiest hallucination to miss is not a novel object. It is a plausible but stale memory, such as believing the mug is still on the counter after another agent already moved it.

A common assumption is that giving the LLM a longer context window, or prompting it more carefully, will fix embodied hallucination. This is wrong because the root cause is not a language problem: the model's context was accurate when it was written, then the physical scene changed and the context was not updated. No amount of prompt engineering refreshes a stale observation. The correct mental model is that embodied memory is a state-estimation problem: every world-state fact must carry a timestamp and a freshness threshold, must be re-verified by sensors before a high-consequence action, and must be expired rather than carried silently across replanning cycles.

Practical Example

A household robot may remember that the user prefers tea in the blue mug across many days, but it should not remember that the blue mug is on the left shelf unless that fact was refreshed by recent perception. One memory is durable preference; the other is dynamic scene state.

Real-World Application: warehouse fulfillment

Amazon's robotic fulfillment centers run hundreds of mobile drive units that share aisles with human pickers and other robots. The fleet controller treats each shelf-pod location as freshness-limited world state: a pod marked "available" is re-confirmed by floor sensors before a drive unit commits to a path, because a co-worker or another robot may have moved or occupied it since the last map update. Acting on a stale pod location would route a robot into an occupied cell, exactly the synchronization failure this section describes.

Memory Hook

Research Frontier

Three active lines of work are pushing embodied memory beyond the stale-context problem. First, persistent 3D semantic memory with LLM querying: systems such as OpenFMNav (Kuang et al., 2024, arXiv 2409.03992) and SpatialBot (Cai et al., 2024) build spatially indexed object maps that survive across episodes; the LLM planner issues natural-language queries against the map rather than against its context window, so stale context cannot override fresh geometry. The active research question is how to merge map updates from multiple traversals without accumulating ghost objects. Second, online world-model verification before action execution: work from Google DeepMind's robotics group and the RT-2 follow-on line (2024-2025) suggests that a lightweight video-prediction head may flag when the observed pre-action frame diverges from what the planner expected, halting execution before a grasp commits rather than after; as of 2025, the open challenge is keeping prediction latency below one replanning cycle on embedded hardware, and no single consolidated benchmark yet quantifies this gain across platforms. Third, calibrated hallucination detection for long-horizon plans: EMBODIED-RAG (Min et al., 2024, CoRL 2024, where RAG stands for retrieval-augmented generation, the practice of fetching relevant stored records before generating a response instead of relying on the model's parameters alone) demonstrates retrieval-augmented planning where each retrieved memory item carries a confidence score derived from the cosine similarity (a measure between -1 and 1 of how closely two numeric vectors point in the same direction, used here to score how well a memory matches the current query) between the query embedding and the stored observation embedding, giving the planner a principled signal for when to request a sensor refresh rather than acting on a low-confidence recall. An open problem well-suited to a PhD dissertation: current freshness-gating schemes treat time as the only staleness signal, but object volatility varies; a cup is stable for hours while a closing gripper makes neighboring objects volatile in milliseconds. Designing a volatility-aware memory expiry model that learns per-object-class and per-action-type expiry rates from trajectory data, and integrating it with a replanning loop that selectively refreshes only the volatile subset of memories before each action, remains unsolved and directly limits deployment of LLM planners in dynamic shared workspaces.

Self Check

Can you list one fact in your system that should persist across sessions and one that should expire within seconds unless perception reconfirms it?

This links to classical filtering and simultaneous localization and mapping (SLAM): language memories and symbolic task facts must obey the same belief-management discipline as geometric state. Without it, the planner weights a ten-minute-old caption and a ten-millisecond-old sensor reading equally.

That is also why hallucination should be decomposed. A model may hallucinate semantically, but many embodied 'hallucinations' are actually stale-state propagation errors, not fabricated facts. Better memory schemas, not bigger models, are often the right fix.

Naming stale-state propagation as the real culprit matters because its consequences in the physical world are far harsher than in text. Why stale-state propagation matters physically: a software agent that acts on outdated information returns a wrong answer. A robot that acts on outdated information applies force to the wrong place. Grasping empty space strains the wrist joint; commanding a mobile base toward a now-occupied aisle causes a collision. The physical world does not tolerate the quiet retry that a language model can regenerate at no cost. Irreversible motion makes memory freshness a safety constraint, not merely an accuracy concern.

The mechanics of stale-state propagation

How propagation happens mechanically: the planner writes facts to the context window once per replanning cycle. Those facts persist verbatim until the next cycle. Any action the planner selects between cycles therefore treats those facts as current, even after the scene has changed. The attention mechanism carries no freshness signal, so a stale fact receives the same logit weight (the raw, pre-probability score the model assigns to a token or fact before ranking it against alternatives) as a fresh one. The propagation path follows four steps: the planner writes an observation to context at $t_k$; the scene changes at $t_k + \delta$; the planner reads that same context at $t_{k+1}$ without detecting $\delta$; and the action executes against the old state. A typical LLM replanning cycle runs at roughly 1 to 2 Hz, that is, a cycle length of about 500 to 1000 milliseconds, and in practice this varies with model size and hardware. A human co-worker can displace an object in under 300 milliseconds. One replanning gap is therefore typically one and a half to three times longer than the fastest physical change in the shared workspace, enough for the scene to change at least once before the planner looks again.

Think of a navigator using a paper chart marked in pencil. Each time the ship takes a position fix, the navigator erases the old dot and draws a new one. If the navigator stops updating the chart but keeps steering by it, every course correction is computed against a position the ship occupied minutes ago. The longer the gap between fixes, the further the pencil dot drifts from reality, and a confident, mathematically correct heading plotted from that dot still drives the ship onto rocks. The LLM context window is that pencil dot: perfectly accurate when written, silently wrong the moment the world moves on without a new fix.

Code Fragment 2 stores a memory record with provenance and freshness. This is the minimum structure needed to talk coherently about embodied hallucination instead of merely complaining that the agent 'made something up.'

  1. Tag each memory by type: preference, world state, task progress, or explanation.
  2. Attach timestamps and evidence sources to every remembered fact.
  3. Force memory retrieval to pass through a fact-validity gate before execution.
  4. Record contradiction events when perception and memory disagree.
  5. Evaluate memory systems on tasks with delayed execution and hidden state changes.
# Build one provenance-rich memory record and gate it before allowing a grasp.
# The record names its evidence source and age so a stale claim cannot pass silently.
from dataclasses import dataclass
@dataclass
class MemoryRecord:
fact: str
source: str # e.g. the camera frame or sensor reading that produced this fact
age_s: float # seconds since the fact was observed
fact_type: str # "preference", "world_state", "task_progress", or "explanation"
def is_actionable(record: MemoryRecord, max_age_s: float) -> bool:
if record.fact_type == "preference":
return True # durable facts do not expire
return record.age_s <= max_age_s
record = MemoryRecord(
fact="red_mug_at_counter_pos_3",
source="camera_frame_104",
age_s=52.0,
fact_type="world_state",
)
print(is_actionable(record, max_age_s=5.0))
False

The expected output is a provenance-rich memory record that blocks direct action because the scene fact is too old. This is exactly the kind of trace you want before calling a behavior a hallucination, since the deeper mechanism is often stale world state rather than fabricated semantics.

Code Fragment 2: This MemoryRecord keeps provenance and freshness visible. The planner can see that the fact came from camera_frame_104 52 seconds ago and is too old for direct execution, which is a much sharper diagnosis than the generic label 'hallucination.'

When memory-rich agents fail, check whether the wrong fact was retrieved, whether the fact was stale, or whether the verifier failed to challenge it. Those paths lead to very different architectural fixes.

Key Takeaway

Embodied memory is valuable only when it behaves like a state-estimation aid rather than an untyped bag of text. Concretely, that means you now have the three things promised at the start of this section: a way to audit what an agent remembers (the provenance and type tags on each memory record), a rule for when each fact should expire (the freshness threshold $\tau$ checked against age $\delta_i$), and a gate that catches hallucinated or stale state before it reaches a physical action (the confidence-and-freshness check in step 5 of the algorithm).
