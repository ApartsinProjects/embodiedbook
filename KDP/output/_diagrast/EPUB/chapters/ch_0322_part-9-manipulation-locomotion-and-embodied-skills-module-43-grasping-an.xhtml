Big Picture

What This Section Develops

This section develops grasp synthesis as a concrete embodied AI skill rather than a label. The core contract is: observe object point cloud, candidate contact pairs, friction cone, and uncertainty, score grasps and choose a robust approach pose, and judge the result with grasp quality and held-out object success.

Action Is The Test

Dexterity begins when the robot treats contact as information, not only as a way to hold an object. For Grasp synthesis: analytic and learned (Dex-Net lineage), the decisive question is whether the loop can recover from the highest geometric score fails because the sensor uncertainty was ignored.

Theory

The minimal formal object is a partially observed control loop. At time $t$, the agent receives $o_t$, estimates $\hat s_t$, chooses action $a_t$, and receives a new observation $o_{t+1}$. In Grasp synthesis: analytic and learned (Dex-Net lineage), the important state variables are object point cloud, candidate contact pairs, friction cone, and uncertainty, while the action interface is to score grasps and choose a robust approach pose.

The method is useful only if it improves a construct-matched measurement: grasp quality and held-out object success. That metric must be co-computed for the baseline and the library shortcut on the same scenario panel, with the same seed policy and the same termination rules.

Mechanism

The mechanism is not a black box. It is a chain of observation encoding, state estimation, constraint checking, action selection, execution, and verification. Each link should emit a log record so a failure can be traced to perception, state, planning, control, timing, data coverage, or evaluation.

Common Failure Mode
