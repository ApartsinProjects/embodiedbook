Section 51.4: Continual learning and catastrophic forgetting

"Continual learning and catastrophic forgetting matters when the next action changes the evidence you thought you had."

A Careful Control Loop
Big Picture

Continual learning and catastrophic forgetting is the adaptation without erasure lens for open-world and lifelong embodiment. Continual learning asks the agent to improve on new experience without destroying old competence. In robotics, forgetting can become a safety problem, not just a lower benchmark score.

The key question is practical: What changes, what must be retained, and which regression panel blocks an unsafe update?

Worked Example

Consider a grasping policy updated on new packaging. It should learn the new box shape without losing the stable grasps it already had for cups, tools, and fragile objects.

# pip install gymnasium
import gymnasium as gym

env = gym.make("CartPole-v1")
obs, info = env.reset(seed=7)
for step in range(5):
    action = env.action_space.sample()
    obs, reward, terminated, truncated, info = env.step(action)
    print(step, action, reward, terminated or truncated)

Practical Recipe

  3. Add the library implementation only after the baseline behavior is understood.
  4. Record failures as structured cases: perception error, state error, planning error, control error, or evaluation error.
  5. Run at least one perturbation test before trusting the result.
Common Failure Mode

The common mistake in Continual learning and catastrophic forgetting is to celebrate the component score before checking the closed-loop handoff. The failure usually appears at the boundary: stale state, wrong frame, delayed action, saturated actuator, or metric that ignores the real task cost.

Practical Example

A continual-learning record should include old-task baseline, new-task improvement, replay source, update rule, retained score, safety failures, and rollback version. Wins belong in the chapter only when old and new metrics are co-computed on one config.

  1. Write a one-paragraph task contract with observation, action, success, and failure fields.

Failure Analysis Pattern

When Continual learning and catastrophic forgetting fails, avoid labeling the whole method as weak. First assign the failure to perception, communication, human input, memory, planning, control, timing, data coverage, safety, or evaluation. Then rerun one controlled perturbation that isolates the suspected cause. This pattern turns a disappointing rollout into a reusable diagnostic asset.

Formal Object

$\theta_{t+1}=\theta_t-\eta\nabla_\theta\big(\ell_{\mathrm{new}}+\beta\,\ell_{\mathrm{retention}}\big),\quad \ell_{\mathrm{retention}}=\|\theta-\theta^\star\|_{\Omega}^2$

Continual learning is not just about getting better on the next task. It is an optimization problem with a retention term that protects previously useful behavior. Catastrophic forgetting is what happens when the gradient signal from the new task overwhelms that retention mechanism.

Retention-aware adaptation loop
  1. Keep a reference snapshot or sufficient statistics for important old-task parameters.
  2. Update on the new task while penalizing changes to parameters with high old-task importance.
  3. Replay old experiences or distilled summaries on every adaptation window.
  4. Trigger rollback or slower learning when old-task metrics cross a retention threshold.

# Check whether adaptation harmed old skills.
scores = {
    "before_new_task": {"old_task": 0.88, "new_task": 0.21},
    "after_adaptation": {"old_task": 0.63, "new_task": 0.74},
}

old_drop = round(scores["before_new_task"]["old_task"] - scores["after_adaptation"]["old_task"], 2)
print("old_task_drop", old_drop)

A 0.25 drop is large enough that a deployment team should ask whether the adaptation is acceptable, whether replay coverage is weak, or whether a slower modular route is needed. Continual learning is only successful when the gain on the new task justifies the retention cost.

Failure Mode To Test

Continual learning fails when only the new task is monitored during adaptation. In embodied systems that can quietly erase safety reflexes or navigation habits that were never included in the adaptation objective.

Key Takeaway

Continual learning is useful only when adaptation and retention are measured together.
