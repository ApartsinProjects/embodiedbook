Action Is The Test

Dexterity begins when the robot treats contact as information, not only as a way to hold an object. For Sim-to-real for dexterity, the decisive question is whether the loop can recover from a policy survives randomized visuals but fails on real friction and compliance.

Theory

The minimal formal object is a partially observed control loop. At time $t$, the agent receives $o_t$, estimates $\hat s_t$, chooses action $a_t$, and receives a new observation $o_{t+1}$. In Sim-to-real for dexterity, the important state variables are domain randomized geometry, friction, latency, compliance, and sensor noise, while the action interface is to train with variability and validate on real perturbation panels.

The method is useful only if it improves a construct-matched measurement: real transfer success and failure taxonomy coverage. That metric must be co-computed for the baseline and the library shortcut on the same scenario panel, with the same seed policy and the same termination rules.

Mechanism

The mechanism is not a black box. It is a chain of observation encoding, state estimation, constraint checking, action selection, execution, and verification. Each link should emit a log record so a failure can be traced to perception, state, planning, control, timing, data coverage, or evaluation.
