Chapter 44: Tactile and Visuo-Tactile Learning

"An agent becomes interesting at the exact moment the world refuses to be a dataset."

A Patient Embodied AI Agent
Big Picture

Tactile and Visuo-Tactile Learning matters because embodied intelligence is a closed loop. The agent must turn partial observations into useful state, choose actions under uncertainty, and learn from the consequences in a physical or simulated world.

Remember This Chapter

Touch gives the robot evidence that vision often cannot see. Slip, pressure, shear, texture, and compliance turn contact from a hidden variable into a learnable signal.

Chapter Overview

Chapter 44 develops Tactile and Visuo-Tactile Learning through a concrete build path: formalize the state and action interface, build a minimal baseline, replace the fragile part with maintained tools, then evaluate failures on a shared scenario panel.

The practical stack emphasizes DIGIT and GelSight, AnySkin and ReSkin style sensors, TACTO and Tactile Gym, LeRobot, MuJoCo touch sensors. The theory thread keeps the mechanism visible: what the agent observes, which physical constraints shape action, and which metric proves that the loop is recoverable.

Prerequisites

Readers should be comfortable with coordinate frames, control loops, simulation, robot data, and same-panel evaluation. This chapter links back to those foundations whenever tactile robot learning depends on them.

Hands-On Lab: Build the Chapter System

Duration: about 90 to 150 minutesDifficulty: Intermediate to Advanced

Objective

Train a slip detector from synchronized RGB, tactile, and gripper state. Evaluate it on held-out objects, worn sensor skins, and changed lighting.

Steps

  1. Write the observation, action, metric, recovery, and safety contract.
  2. Implement a transparent baseline that exposes the mechanism.
  3. Run the maintained tool route with the same scenarios and metric code.
  4. Log slip precision, slip recall, contact onset latency, object success, and cross-sensor generalization in one artifact.
  5. Write a postmortem that explains two failures and one reliable behavior.
